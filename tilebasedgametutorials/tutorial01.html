More Graphics

In this example you can see that I added some new tile graphics to the map. Even though they're still a very dismal gray, the fact that they're not all square makes the level more appealing to look at. It's also interactive via mouse or touch, so check it out:

canvas

The purpose of this example is really just to show you how the index relates to column and row, but if you want an explanation on blitting multiple graphics to the screen, here it is:

First of all, blitting isn't a real word, but it stands for bit block transfer and what it refers to is transferring image data from one surface to another. In this case, individual tile graphics are blitted from the "source" tile sheet image to the "destination" html5 canvas. If you recall how to calculate the destination x and y position of a tile from the last example, you already know how to calculate the source x and y position of a tile graphic. Here's a section of the code to show you just how similar the two are: 

var destination_column = index % this.columns;
var destination_row = Math.floor(index / this.columns);
var destination_x = destination_column * tile_sheet.tile_width;
var destination_y = destination_row * tile_sheet.tile_height;

var source_column = value % tile_sheet.columns;
var source_row = Math.floor(value / tile_sheet.columns);
var source_x = source_column * tile_sheet.tile_width;
var source_y = source_row * tile_sheet.tile_height;

As you can see, they're basically the same. The only difference is that destination uses "index" which refers to a unique space on the tile map and source uses "value" which refers to a unique space in the tile sheet. The actual blitting is then done by drawImage using the source and destination information.

So, that's how it's done. I'd write some more about it, but there's really not much to say. As always, you have full access to the commented source code so check it out.
